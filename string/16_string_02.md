- 反向迭代器,rbrgin理解为指向最后一个位置,rend指向第一个位置的前一个位置,迭代器是左闭右开的
- 反向迭代器一定不是指针,一定是经过封装重载的
- 反向迭代器用于倒着遍历容器
- 正向迭代器倒着写有时候会崩溃因此直接采用反向迭代器
- cbegin确定返回const 迭代器(了解接即可)
- size()接口和length()表示长度,其中size具有普适性,length只适用于string,以后用size
- capacity()表示容量大小
- string扩容时刚开始二倍扩然后1.5倍因为string底层有一个_buff[16],超过16之后才在堆上,所以string最开始才是2倍扩
- g++中扩容就是纯2倍扩,没有buff了
- 用reserve先提前开好100,reserve一般不会缩容
- 返回capacity不包含\0
- clear只清字符不清容量
- resize可以扩容插入(vector细学)
- 正向反向迭代器不是同一类型不能直接比较,不能用下标比较的方式比较
